# Offer Calculation — Real API Test Report

> Generated by running the **exact offer-resolution algorithm** from
> `src/Service/Order/OrderService.ts` → `calculateOrderOffers` (the same code the
> `POST /order/preview` and `POST /order` endpoints call), driven by **real product
> prices pulled live from the production database**.
>
> **No orders were created.** `/order/preview` is side-effect-free and shares the
> identical pricing code with `createOrder`, so these totals are exactly what a real
> purchase would charge — without touching stock or emailing admins.
>
> Shipping cost = EGP 95.00 (the real value from the reported order).
> Per-item price = `finalPrice ?? price` (the live sale price).

## How the engine works (2 stacking layers)
1. **Flash sale** (homepage): specific products get a % off. Flash items are **excluded**
   from cart-offer qualification (their price/qty don't count toward thresholds) but their
   discounted price IS charged.
2. **One cart offer per order**: every active cart offer is evaluated against the
   **non-flash** items; the single offer that saves the customer the **most** is applied.
3. **Final total** = `subTotal (flash-discounted) − cartDiscount + (freeShipping ? 0 : shipping)`.
